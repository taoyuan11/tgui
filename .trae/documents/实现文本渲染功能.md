# 文本渲染功能实现计划

## 1. 分析当前问题
- 按钮组件已实现，但文本未能显示
- 渲染器中 `DrawCommand::FillText` 为空实现
- 项目已依赖 `cosmic-text` 库，可用于文本渲染

## 2. 实现步骤

### 2.1 修改 Canvas 状态和命令
- 在 `CanvasState` 中添加字体相关字段：字体名称、字体大小、字体样式
- 更新 `DrawCommand::FillText` 和 `StrokeText` 以包含字体信息

### 2.2 实现文本渲染核心功能
- 在 `Renderer` 中添加文本渲染所需的资源：
  - 使用 `cosmic-text` 创建 `FontSystem`
  - 实现文本布局和测量功能
  - 将文本渲染为纹理
- 实现 `render_fill_text` 和 `render_stroke_text` 方法

### 2.3 完善 Canvas2D 文本相关方法
- 添加字体设置方法：`set_font`, `set_font_size` 等
- 更新 `fill_text` 和 `stroke_text` 方法以使用当前字体设置

### 2.4 优化按钮文本居中对齐
- 修改按钮的 `draw` 方法，实现文本的精确居中
- 考虑不同字体大小和文本长度的情况

### 2.5 测试与优化
- 测试中英文文本渲染
- 测试不同字体和大小的效果
- 测试不同长度文本的显示
- 优化渲染性能，避免卡顿

## 3. 技术要点
- 使用 `cosmic-text` 库处理文本布局和渲染
- 实现字体选择机制，支持系统默认字体和用户指定字体
- 确保文本在按钮内正确居中对齐
- 优化渲染性能，避免频繁创建资源

## 4. 预期结果
- 按钮文本能够正确显示
- 支持中英文文本
- 文本在按钮内居中对齐
- 支持字体选择和大小调整
- 渲染性能良好，无明显卡顿
